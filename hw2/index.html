Overview

In this homework we explored 3-dimensional geometric modeling to create and modify meshes of various objects.

In section 1, we implemented the basic algorithm to create the mesh objects. First, we created smooth 2-dimensional Bezier curves with the de Casteljau algorithm. Then, we expanded upon the 2-dimensional Bezier curves to create 3-dimensional Bezier surfaces. Both Bezier curves and Bezier surfaces relied on the recursive nature of the de Casteljau algorithm to optimally create the line geometry. From here, we could create surfaces for the mesh.

In section 2, we implemented features to our mesh edit program to improve the rendering of the mesh objects. First, we implemented Phong shading with area-weighted vertex normals. In this algorithm, we derived the unit vector for the weighted sum of every normal vector that is an adjacent face to a specific vertex. This implementation allowed the meshes to appear smoother. Then, we implemented edge flipping and edge splitting to provide more control to change the triangles within the mesh. Edge flipping moved the edge whereas edge splitting added a new vertex and new edges to add more detail to the mesh. Lastly, we implemented loop subdivision for mesh upsampling which combines all previous features to convert the mesh to higher-resolution. using the original mesh data, extra elements were added to reinforce the details of each triangle of the mesh, producing much smoother meshes.

Overall, this homework introduced features to improve mesh generation.

Part 1

The de Casteljau's algorithm is used to evaluate Bezier curves, given a set of control points and a parameter t. Given n control points, the algorithm starts by linearly interpolating (n - 1) intermediate control points with respect to the t parameter, a decimal between 0 and 1. So, given a set of control points P1, P2, … , PN, we can calculate the ith control point PI' at the next subdivision as follows:

We can apply this algorithm recursively on the resulting set of control points until there is only one control point remaining.

For this assignment, the separable 1D case of the algorithm was implemented in one function, the 2DVector case of the evaluateStep() function. This function takes in a list of control points, represented as 2D vectors. It then iterates through the first (n - 1) points, lerping the ith and (i+1)th point and storing the result in an array. Once all points have been processed, the function returns the array containing the (n - 1) control points of the next subdivision level.

Part 2

Briefly explain how the de Casteljau algorithm extends to Bezier surfaces and how you implemented it in order to evaluate Bezier surfaces.

In the following part, we extend the 1D separable case of de Casteljau's algorithm to 2D to generate Bezier surfaces. This is done by calculating Bezier curves with respect to the u parameter for each control point, retrieving the final control point of each respective curve, and then calculating the final position on the Bezier surface by calculating the Bezier curve of the final u-control points with respect to v.

Pictured below is a visualization of how the 1D separable case of the algorithm can be extended to 2D. Note that the blue curve represents the final u-control points, and the black point is the final control point.

This implementation required the creation of three functions. The first is evaluateStep(), which takes in a list of 3D control points and the decimal parameter t, and returns the (n - 1) control points of the next subdivision level. These points are generated by lerping intermediate points together in the same manner as described in Part 1.

The next function is evaluate1D(), which takes in a list of 3D control points and a parameter t, and recursively evaluates de Casteljau's algorithm on the list using evaluateStep(). The function returns when there is only one control point in the list and returns said point.

The final function is evaluate2D(), which takes in two parameters u and v, and returns the point on the Bezier surface corresponding to the point (u, v) on the surface. This is done by finding all u-points on the surface, which is done by lerping the control points with respect to the parameter u. This functions identically to the evaluateStep() function from part 1. Once an array of u-points is generated, we return the final point evaluated by the evaluate1D() function, when passed in the u-points as control points and v as the parameter.

For the area-weighted vertex normals algorithm, we initialize a vector, called weighted_sum, as <0, 0, 0> to store the sum of the vectors normal to each face that contains the vertex. Then, we loop through all of the halfedges belonging to faces that contain the given vertex. To do this, we start with this vertex's halfedge, called h, and visited h's twin's next halfedge to ensure that every iteration's halfedge belonged to a new face with the given vertex. The loop ends when halfedge h has finished looping around all halfedges and has come back to the starting halfedge. The following diagram shows the iterated halfedges:

Inside of the loop, we calculate the normal vectors by doing a cross product on this vertex's halfedge and the next halfedge. These halfedges are converted to vectors by calculating the positions of all 3 vertices a, b, and c belonging to the face triangle. We define vertex a as the given vertex, determine vertex b by getting the next halfedge's associated vertex, and determine vertex c by getting the next's next haldedge's associated vertex. The halfedge vector is the position difference between vertex b and vertex a. The next halfedge vector is the position difference between vertex c and vertex b. Part 4

For the edge flip, we defined all current halfedges, vertices, edges, and faces:

  • Halfedges: halfedge_b_c, halfedge_c_a, halfedge_a_b, halfedge_c_b, halfedge_b_d, halfedge_d_c
  • Vertices: b, c, a, d
  • Edges: edge_b_c, edge_c_a. edge_a_b, edge_b_d, edge_d_c
  • Faces: face_bca, face_cbd

After these definitions, face_bca and face_cbd are checked for boundary edges. If so, then the given edge pointer e0 is returned and nothing is changed.

Then, all edges and faces are set to their new corresponding halfedge.

Through pointer reassignment, halfedge_b_c is changed to represent halfedge_a_d and halfedge_c_b is changed to represent halfedge_d_a.

The given edge pointer e0 is defined as edge_b_c and changed to represent edge_a_d. edge_a_d is given halfedge_a_d as its corresponding halfedge. The rest of the edges did not change their corresponding halfedge.

face_bca is changed to represent face_adc and face_dab is changed to represent face_cbd. face_adc is given halfedge_c_a as its corresponding halfedge and face_cbd is given halfedge_d_a as its corresponding halfedge.

Lastly, all halfedges were reassigned next, twin, vertex, edge, and face values based on the new arrangement. After that, the given edge pointer e0 is returned with all the changes done.

We did not encounter many bugs at all since we were careful to clearly specify which halfedges, vertices, edges, and faces would have their representations changed with new variables as intermediary elements. For example, a new pointer halfedge_a_d was created and set to halfedge_b_c because halfedge_b_c represents halfedge_a_d after flipping the edge.

Part 6

After implementing the flipEdge and splitEdge functions in the mesh edit simulation, we can now implement loop division for mesh upsampling, a common subdivision rule for triangle meshes. The algorithm itself is simple: 4-1 subdivide each triangle in the mesh, and then assign new vertex positions determined by a weighted sum of the vertices surrounding it. A visual of what 4-1 subdivision entails is pictured below:

Note that this operation requires the splitEdge and flipEdge operations described above.

Below is a more in-depth description of this project's implementation of loop subdivision:

  1. Traverse through all vertices in the mesh and calculate their new vertex position in the upsampled mesh using the following rule:
    1. new_vertex_position = (1 - n * u) * original_vertex_position + u * original_neighbor_position_sum
      1. Where n is the degree of the current vertex, u is the scaling value, original_vertex_position stores a 3DVector storing the vertex's original position, and original_neighbor_position_sum is a 3D vector storing the sum of the positions of the neighboring vertices.
      1. Note that u is ⅜ if n is 3, and 3/ (8 * n) otherwise.
    1. Store this new position in the newPosition attribute of the vertex, and mark all vertices in the mesh as “old”.
  1. Traverse through all edges in the mesh and calculate the new position of each vertex at the midpoint of each, and store this value in the newPosition attribute of the respective edge. Mark each edge traversed as “old”.
    1. The formula for calculating this new position is as follows, with respect to the image below:
      1. 3/8 * (A + B) + 1/8 * (C + D)
      1. Note that this calculation is effectively a weighted sum.
  1. Split all edges in the mesh that have old vertices on both sides. When splitting edges, mark all newly created edges by setting the isNew attribute to “true”.
  1. Flip all new edges in the mesh that are connected to an old and new vertex. Whether or not an edge or vertex is new can be seen by checking the respective isNew attribute. Note that these new edges that will be flipped correspond to the blue edges in the photo below.
  1. Finally, update all vertex positions with their new positions calculated in steps 2 and 3, and set their isNew attribute to “false” to ensure proper behavior on the next subdivision.

As seen above, one very helpful attribute featured in this implementation of the loop subdivision algorithm is the isNew attribute, which the Edge and Vertex data structures (technically EdgeIter and VertexIter) have. This allows us to mark whether vertices and edges are part of the original mesh or not, which is crucial in edge splitting and edge flipping the correct edges. The newPosition attribute shared between both data structures was equally helpful, as it allowed us to re-assign all vertices in the mesh their new positions in the subdivided regime at the very end of the loop subdivision operation without changing their position attribute (which stores a 3DVector tracking their current position). The usage of these two attributes ensured less memory usage (due to not needing to create any new edges and only one new variable in upsample()) and a much faster runtime.
