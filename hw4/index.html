Overview

In the following project, we simulated real-time cloth movement using a mass and spring based system, and implemented a variety of shaders.

In part 1, we implemented a physics-based system to simulate a physical cloth to interact in an environment. To do this, we created a system of point masses, to represent physical mass, and springs, to represent connections between point masses. We evenly spread out the point masses and used structural, shearing, and bending springs to mimic different kinds of material behavior.

In part 2, we simulated physically-based motion through numerical integration. To be more specific, we implement physical-based motion equations in the simulate function in order to apply the appropriate amount of force to each point mass in the cloth, given some timestep. To do this, we first calculate the total force applied to each point mass in the cloth via Hooke’s Law. We then use Verlet integration to calculate the position changes of all point masses. Finally, we constraint all position updates as per the method described in the SIGGRAPH 1995 Provot paper “Deformation Constraints in a Mass-Spring Model to Describe Rigid Cloth Behavior.

In part 3, we add in collision detection support, based on whether the cloth is colliding with a sphere or itself. We first implement sphere collision detection, where we detect whether a cloth’s point masses intersect a sphere, and adjust the point mass positions accordingly. We then implemented plane intersection support in a similar fashion, checking to see if a point mass at a given time point intersects the plane, and adjusting accordingly if so.

In part 4, we implemented self-collision to add further realism to the cloth physics simulation. Having added plane and sphere collision properties, self-collision was necessary to prevent the cloth from bunching up together in an extremely tight space. As a constraint, we used 2 * thickness as for the minimum distance between point masses. To optimize the simulation, we employed a hashing function based on 3D spaces comprising the cloth to minimize the amount of particle adjustment calculations. To do these corrections, we get the hashed key and only look at the point masses within the 3D space. All point masses with distances less than 2 * thickness then have their position moved to meet this constraint. In doing so, the cloth could now collide with itself.

In part 5, we implemented shading to the cloth simulation. However, instead of using pure CPU processing, we implemented the shading with GLSL to utilize the parallel processing of the GPU. We had the opportunity to implement shading and texturing that are similar to ones we have implemented on previous homework assignments, including Diffuse Shading, Blinn-Phong Shading, and Texture Mapping. Furthermore, we had the opportunity to implement new texturing methods such as Bump Mapping, Displacement Mapping, and Reflections.

Overall, this assignment allowed us to explore how the physics behind cloth simulations are implemented, and how shaders are programmed and applied.

Overall, increasing ks results in a stiffer cloth.

Part 5 - Cloth Sim

A shader program (typically written in GLSL) are GPU-based programs that run for different stages in the graphics pipeline. They are isolated programs designed to run in parallel, greatly increasing rendering speed. Although there are many shader stages in the OpenGL rendering pipeline, the ones used in this project were the vector and fragment shaders respectively.

Vector shaders are applied first in the pipeline and process vertices via transformations, often changing the positions, normal vectors, etc. and writing final positions. To be more specific, vertex shaders write the final gl_Position value for each vertex. Note that these values calculated by the vertex program (such as the vertex’s normal, position, etc.) are then used by the fragment shader. Fragment shaders are applied further down the pipeline after rasterization, and produce “fragments” from pixel samples from the vertex values outputted by the vertex shader (which are interpolated via barycentric coordinates before being passed in).  These fragments are then processed by the shader to compute write per-fragment values and output per-fragment colors.

We implemented the Blinn-Phong shader model in this project. The Blinn-Phong reflection model is a simple shading model, with three components, pictured below:

The light color for each fragment / pixel sample returned by this algorithm is the sum of the

ambient light, the diffuse light, and the specular light at that fragment, represented as La, Ld, and Ls respectively.

Ambient lighting refers to any shading that depends on no external factors aside from the external illumination. It is often a constant color scaled by a constant ka, the ambient coefficient, which is manually chosen. In this project, the illumination parameter Ia was also manually chosen. La then represents the color of the ambient light at the fragment.

Diffuse lighting refers to shading that does not depend on the view/camera direction. It consists of the illumination from the light source (the illumination of the light source scaled by its distance from the point of intersection) scaled by the diffuse coefficient (manually determined) and the angle between the normal of the current fragment/point of intersection and the light ray emitted from the light source. Note that we do not consider negative values when taking the dot product, as negative colors don’t make sense for our model, and this would imply that the normal and light source vectors are pointing in opposite directions, indicating no diffuse shadowing. Thus we take the max between this dot product and 0, and scale the illumination by it.

Specular lighting refers to specular highlights on an object, where the intensity of this lighting is dependent on the viewing/camera angle. To calculate specular lighting, we multiply the illumination from the light source by ks, the specular coefficient (manually chosen to increase/decrease shininess / intensity), and the max of the dot product between the normal vector of the surface at the intersection point and the halfway vector h, which is the vector that bisects the light ray and the view ray. Note that we take the max between 0 and this dot product for similar reasons as outlined above, and then take this quantity to the pth power, where p is manually chosen; a higher p will increase the intensity of the specular light.

Finally, we sum of all of these components. This is the final color value calculated for the fragment passed into our Blinn-Phong shader. We calculate this value for each fragment to apply Blinn-Phong shading to the simulation.
